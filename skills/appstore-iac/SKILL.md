# App Store IaC — Full App Store Connect Sync

One command to push your entire App Store Connect configuration.

`/appstore-iac apply` syncs: app info, categories, age rating, subscriptions, pricing, listing metadata, screenshots, review details, and export compliance.

For local StoreKit sandbox testing only, use `/storekit-iac generate-storekit`.

## Reference

Load these before starting:
- [Shared ASC API](../shared/asc-api.md) — JWT auth, error handling, rate limiting, character limits, hard-won lessons
- [API Reference](api-reference.md) — App metadata, version, localization, review, subscription endpoints
- [Screenshot Upload](screenshot-upload.md) — 4-step screenshot upload flow

## Modes

- No argument or `plan` → **Plan mode** (default, no mutations)
- `apply` → **Apply mode** (pushes changes after confirmation)

## Config Files

All read-only inputs. The skill never modifies them.

| File | Required | Generated By |
|---|---|---|
| `appstore/app-info.json` | Yes | Manual or generator |
| `appstore/listing.json` | Yes | `ios-store-listing` |
| `appstore/products.yml` | Yes | Manual (subscription IaC) |
| `appstore/age-rating.json` | Yes | `ios-privacy` |
| `appstore/review-notes.json` | Yes | `ios-privacy` |
| `appstore/export-compliance.json` | Yes | `ios-privacy` |
| `appstore/privacy-labels.json` | Info only | `ios-privacy` (NO API — manual in ASC) |
| `screenshots/appstore/{device}/` | Optional | `ios-screenshots` |
| `appstore/listing-{locale}.json` | Optional | `ios-localize` |

### Validation

Before any API call, validate required generators have run, then check field lengths.

**Implicit generator invocation** — if listing.json or site URLs are missing, offer to run their generators first:

```bash
MISSING_GENERATORS=()

if [ ! -f "appstore/listing.json" ]; then
  MISSING_GENERATORS+=("listing (run /ios-store-listing)")
fi

if ! grep -q '"marketing_url"' "appstore/app-info.json" 2>/dev/null; then
  MISSING_GENERATORS+=("site (run /ios-site init)")
fi

if [ ${#MISSING_GENERATORS[@]} -gt 0 ]; then
  echo "Missing generator outputs:"
  for g in "${MISSING_GENERATORS[@]}"; do echo "  - $g"; done
  echo ""
  read -p "Run these now? [Y/n] " CONFIRM
  if [ "$CONFIRM" = "n" ] || [ "$CONFIRM" = "N" ]; then
    echo "Aborting. Run the generators manually and try again."
    exit 1
  fi
  # Delegate in order: listing first (site landing can reuse listing hero)
  # The skill invokes /ios-store-listing then /ios-site init interactively.
fi

# After generators ran (or if they already existed), verify all required files
for f in app-info.json listing.json products.yml age-rating.json review-notes.json export-compliance.json; do
  test -f "appstore/$f" || echo "MISSING: appstore/$f — run the appropriate generator skill"
done
```

```python
# Validate field lengths

info = json.load(open('appstore/app-info.json'))
```

## Plan Mode

### Step 1: Read all config files

Parse every config file. Extract `app_id` from `app-info.json`.

### Step 2: Generate JWT and fetch current state

See shared/asc-api.md for JWT generation. Fetch in order:

```bash
# App info + localizations + age rating
GET /v1/apps/{app_id}/appInfos?include=appInfoLocalizations,ageRatingDeclaration

# Editable version
GET /v1/apps/{app_id}/appStoreVersions?filter[platform]=IOS

# If version exists: localizations, review detail, screenshot sets
GET /v1/appStoreVersions/{versionId}/appStoreVersionLocalizations
GET /v1/appStoreVersions/{versionId}/appStoreReviewDetail

# Subscription groups
GET /v1/apps/{app_id}/subscriptionGroups
# For each group: subscriptions, localizations, offers
```

### Step 3: Diff and present

```

  = name: "Udana" (30 chars max)
```

## Apply Mode

### Step 1: Run plan and confirm

Show diff. Ask: "Apply these changes to App Store Connect?"

### Step 2: Execute in dependency order

**This order is critical. Do not reorder.**

#### Phase 1: App-Level Config (no version needed)

**1. App Information** — name, subtitle, categories
```
GET /v1/apps/{app_id}/appInfos
PATCH /v1/appInfos/{appInfoId}                    → categories (as relationships)
PATCH /v1/appInfoLocalizations/{locId}            → name, subtitle, privacyPolicyUrl
```

**2. Age Rating**
```
GET /v1/appInfos/{appInfoId}/ageRatingDeclaration
PATCH /v1/ageRatingDeclarations/{ageRatingId}     → all rating fields
```
Map age-rating.json keys to API attribute names (see api-reference.md mapping table). Set `healthOrWellnessTopics: true` for health/fitness apps.

**3. App Availability**
```
POST /v1/appAvailabilities                        → territories
```
For a free app, no pricing needed.

#### Phase 2: Subscriptions (from products.yml)

**4. Subscription Groups & Products**
Run the same logic as storekit-iac apply:
- Create/verify subscription groups (`POST /v1/subscriptionGroups`)
- Create/verify subscriptions (`productId` camelCase!)
- Create/update localizations (max 55 chars description)
- Set subscription availability (MUST be before pricing)
- Set subscription prices (omit attributes for starting price)
- Create introductory offers
- Create promotional offers (duration must match sub period, prices inline)
- Set review notes on subscriptions

See storekit-iac api-reference.md for detailed endpoint docs.

#### Phase 3: Version-Specific Config (requires editable version)

**5. Find editable version**
```
GET /v1/apps/{app_id}/appStoreVersions?filter[platform]=IOS
```
Look for `appStoreState` in: `PREPARE_FOR_SUBMISSION`, `DEVELOPER_REJECTED`, `REJECTED`.
If none found → STOP. Tell user: "No editable version found. Upload a build first."

**6. Version Localizations**
```
GET /v1/appStoreVersions/{versionId}/appStoreVersionLocalizations
PATCH /v1/appStoreVersionLocalizations/{locId}    → description, keywords, whatsNew, promotionalText, supportUrl, marketingUrl
```
For additional locales, `POST /v1/appStoreVersionLocalizations` with `appStoreVersion` relationship.

**7. Screenshots**
Follow screenshot-upload.md for each device directory:
- Create screenshot sets per display type
- Reserve → upload → commit for each image
- Reorder to match filename sort order
- Skip unchanged files (checksum comparison)

**8. Review Details**
```
GET /v1/appStoreVersions/{versionId}/appStoreReviewDetail
POST or PATCH /v1/appStoreReviewDetails           → contactFirstName, contactLastName, contactEmail, contactPhone, notes, demoAccountRequired
```

**9. Export Compliance**
```
PATCH /v1/appStoreVersions/{versionId}            → usesNonExemptEncryption
```

### Step 3: Report

After all steps, show summary:

```
App Store IaC Apply — Complete
──────────────────────────────
  ✓ App info: name, subtitle, categories
  ✓ Age rating: 13 fields updated
  ✓ App availability: USA
  ✓ Subscriptions: 2 products, 1 intro offer, 1 promo offer
  ✓ Listing (en-US): description, keywords, whatsNew
  ✓ Screenshots: 3 uploaded (iphone-6.7)
  ✓ Review details: contact info + notes
  ✓ Export compliance: no encryption
  ⚠ Privacy labels: configure manually in App Store Connect

Ready for /ios-submit
```

### Step 4: Verify

Run plan again. Should show 0 changes (except privacy labels manual note).

## Safety Rules

1. **Plan is default.** Bare invocation runs plan.
2. **Confirm before apply.** Always show diff and wait.
3. **Screenshot deletion warnings.** Removing screenshots needs explicit confirmation.
4. **Version guard.** No editable version → stop with clear message.
5. **Config validation.** Check files exist and field lengths before any API call.
6. **Never modify config files.** Read-only inputs.
7. **Never log secrets.** No JWT tokens, no .p8 content.
8. **Idempotent screenshots.** Compare MD5 checksums, skip unchanged.
9. **Self-healing.** Fix reference docs on every API failure.
10. **Privacy labels reminder.** Always note in output that privacy labels are manual.
